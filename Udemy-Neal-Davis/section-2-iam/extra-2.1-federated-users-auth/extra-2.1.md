# Extra 2.1: Federated Users Authentication

__Introduction__    
Federated Users Authentication in AWS allows users to access AWS resources using credentials from external identity providers (IdPs), such as your company’s directory (like Microsoft Active Directory, Okta, or Google Workspace, Cognito) or social login providers. This means you don’t have to create separate AWS IAM users for each person—instead, you manage users in your existing identity system.
